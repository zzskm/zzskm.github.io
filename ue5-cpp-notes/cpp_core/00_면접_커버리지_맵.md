# 면접 커버리지 맵

> 카테고리 1 보조 문서 | 요청한 10개 면접 축 대응표
> 대상 범위: `cpp_core` 토픽 1~30 + `09_면접_경험_워크북.md`

---

## 사용법

1. 먼저 아래 표에서 대응 토픽을 따라 읽는다.
2. 각 축마다 `개념 설명`, `선택 기준`, `실제 경험` 셋 중 무엇이 부족한지 체크한다.
3. `실제 경험`이 비어 있는 축은 `09_면접_경험_워크북.md`로 답변 초안을 채운다.

## 3. 최종 점검 체크리스트

| 질문 | 확인 기준 |
|------|-----------|

| 선택 기준을 말할 수 있는가? | 포인터/참조, `unique_ptr/shared_ptr`, `vector/list/deque`, `mutex/spinlock`처럼 비교 질문에 답할 수 있어야 한다. |
| 실제 경험으로 연결되는가? | `09_면접_경험_워크북.md`에 최소 5개 축 이상 사례가 채워져 있어야 한다. |
| UE5 맥락으로 바꿔 말할 수 있는가? | 각 문서의 `### 3. 왜 중요한가`와 `### 6. UE5 소스 참고 경로`를 연결해 설명한다. |

## 4. 아직 직접 채워야 하는 것

- 본인 프로젝트에서 `shared_ptr`, raw pointer, pool, lock, container 선택을 실제로 어떻게 썼는지
- 크래시나 성능 병목을 어떤 순서로 분석했고 무엇을 근거로 결론 냈는지
- "왜 그 선택을 했는가"를 대안 비교까지 포함해 말할 수 있는 사례

이 문서는 "무엇을 읽어야 하는가"를 정리한 맵이다. 실제 면접 답변 완성은 `09_면접_경험_워크북.md`에서 마무리한다.
